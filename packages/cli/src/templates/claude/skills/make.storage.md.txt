# Make Storage Class

Generate a new storage class and its test file using the `make:storage` CLI command, then complete the generated code with proper implementation.

## Coding Conventions

- Always explicitly show visibility on class methods and properties (`private`, `public`, `protected`)
- Always use arrow functions when possible, but NOT for class methods (class methods use regular method syntax)
- End type names with the `Type` suffix (e.g., `ExceptionStackFrameType`, `ServiceDataType`)
- Start interface names with the `I` prefix (e.g., `IException`, `IService`, `IAiChat`)
- For Entity class, if property is optional, add `null` to its type.
- For dependencies, use `inject` from `@ooneex/container`. For example:

```typescript
import { inject } from "@ooneex/container";
import { decorator, type ISeed } from "@ooneex/seeds";

@decorator.seed()
export class BookSeed implements ISeed {
  constructor(
    @inject(BookRepository)
    private readonly repository: BookRepository,
  ) {}
}
```

## Important

All commands must be run from the root of the project (not from within a package or module directory).

## Steps

### 1. Run the generator

Run the following command to scaffold the storage class and test files:

```bash
bunx @ooneex/cli@latest make:storage --name=<name> --module=<module>
```

Where `<name>` is the name provided by the user. The `--module` option is optional — if provided, files are generated under `modules/<module>/` instead of the project root. The command will generate:
- `src/storage/<Name>Storage.ts` - The storage class file (or `modules/<module>/src/storage/<Name>Storage.ts` with `--module`)
- `tests/storage/<Name>Storage.spec.ts` - The test file (or `modules/<module>/tests/storage/<Name>Storage.spec.ts` with `--module`)

### 2. Read the generated files

Read both generated files to understand the scaffolded code.

### 3. Complete the storage class

Edit `src/storage/<Name>Storage.ts` to complete the implementation:

- Set the `bucket` property to the appropriate bucket name
- Verify the environment variable names match the project configuration (`STORAGE_<NAME_UPPER>_ACCESS_KEY`, `STORAGE_<NAME_UPPER>_SECRET_KEY`, `STORAGE_<NAME_UPPER>_ENDPOINT`, `STORAGE_<NAME_UPPER>_REGION`)
- Add any additional storage-specific methods if needed

The generated class structure follows this pattern:

```typescript
import { Storage, decorator, StorageException } from "@ooneex/storage";
import type { S3Options } from "bun";

@decorator.storage()
export class <Name>Storage extends Storage {
  protected bucket: string;
  private readonly accessKey: string;
  private readonly secretKey: string;
  private readonly endpoint: string;
  private readonly region: string;

  constructor(options?: {
    accessKey?: string;
    secretKey?: string;
    endpoint?: string;
    region?: string;
  }) {
    super();

    const accessKey = options?.accessKey || Bun.env.STORAGE_<NAME_UPPER>_ACCESS_KEY;
    const secretKey = options?.secretKey || Bun.env.STORAGE_<NAME_UPPER>_SECRET_KEY;
    const endpoint = options?.endpoint || Bun.env.STORAGE_<NAME_UPPER>_ENDPOINT;

    // ... validation throws StorageException if missing ...

    this.accessKey = accessKey;
    this.secretKey = secretKey;
    this.endpoint = endpoint;
    this.region = options?.region || Bun.env.STORAGE_<NAME_UPPER>_REGION || "auto";
  }

  public getOptions(): S3Options {
    return {
      accessKeyId: this.accessKey,
      secretAccessKey: this.secretKey,
      endpoint: this.endpoint,
      bucket: this.bucket,
      region: this.region,
    };
  }
}
```

### 4. Complete the test file

Edit `tests/storage/<Name>Storage.spec.ts` to add meaningful tests beyond the scaffolded ones:

- Keep the existing scaffolded tests (class name, bucket field, getOptions method)
- Add tests relevant to the specific storage class behavior

The generated test structure follows this pattern:

```typescript
import { describe, expect, test } from "bun:test";
import { <Name>StorageAdapter } from "@/storage/<Name>StorageAdapter";

describe("<Name>StorageAdapter", () => {
  test("should have class name ending with 'StorageAdapter'", () => {
    expect(<Name>StorageAdapter.name.endsWith("StorageAdapter")).toBe(true);
  });

  test("should have 'bucket' field", () => {
    expect("bucket" in <Name>StorageAdapter.prototype || "bucket" in Object.getOwnPropertyNames(<Name>StorageAdapter.prototype)).toBe(true);
  });

  test("should have 'getOptions' method", () => {
    expect(<Name>StorageAdapter.prototype.getOptions).toBeDefined();
    expect(typeof <Name>StorageAdapter.prototype.getOptions).toBe("function");
  });
});
```

### 5. Lint and format

Run linting and formatting on the generated files:

```bash
bunx biome check --fix src/storage/<Name>Storage.ts tests/storage/<Name>Storage.spec.ts
```
