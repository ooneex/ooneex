# Make Middleware Class

Generate a new middleware class and its test file using the `make:middleware` CLI command, then complete the generated code with proper implementation.

## Steps

### 1. Run the generator

Run the following command to scaffold the middleware class and test files:

```bash
bunx @ooneex/cli@latest make:middleware --name <name> --is-socket <true|false>
```

Where `<name>` is the name provided by the user. The `--is-socket` option determines whether to generate an HTTP middleware or a WebSocket middleware (defaults to `false` if omitted). The command will generate:
- `src/middlewares/<Name>Middleware.ts` - The middleware class file
- `tests/middlewares/<Name>Middleware.spec.ts` - The test file

### 2. Read the generated files

Read both generated files to understand the scaffolded code.

### 3. Complete the middleware class

Edit `src/middlewares/<Name>Middleware.ts` to complete the implementation:

- Implement the `handler` method with actual middleware logic
- Add request/response transformations, authentication checks, logging, etc.
- Inject any required dependencies via the constructor

**HTTP middleware** generated structure:

```typescript
import type { ContextType } from "@ooneex/controller";
import { decorator, type IMiddleware } from "@ooneex/middleware";

@decorator.middleware()
export class <Name>Middleware implements IMiddleware {
  public async handler(context: ContextType): Promise<ContextType> {
    // Example: Add custom header
    // context.response.header("X-Custom-Header", "value");

    return context
  }
}
```

**Socket middleware** generated structure:

```typescript
import type { ContextType } from "@ooneex/socket";
import { decorator, type IMiddleware } from "@ooneex/middleware";

@decorator.middleware()
export class <Name>Middleware implements IMiddleware {
  public async handler(context: ContextType): Promise<ContextType> {
    // Example: Add custom header
    // context.response.header("X-Custom-Header", "value");

    return context
  }
}
```

### 4. Complete the test file

Edit `tests/middlewares/<Name>Middleware.spec.ts` to add meaningful tests beyond the scaffolded ones:

- Keep the existing scaffolded tests (class name, handler method)
- Add tests relevant to the specific middleware behavior

The generated test structure follows this pattern:

```typescript
import { describe, expect, test } from "bun:test";
import { <Name>Middleware } from "@/middlewares/<Name>Middleware";

describe("<Name>Middleware", () => {
  test("should have class name ending with 'Middleware'", () => {
    expect(<Name>Middleware.name.endsWith("Middleware")).toBe(true);
  });

  test("should have 'handler' method", () => {
    expect(<Name>Middleware.prototype.handler).toBeDefined();
    expect(typeof <Name>Middleware.prototype.handler).toBe("function");
  });
});
```

### 5. Lint and format

Run linting and formatting on the generated files:

```bash
bunx biome check --fix src/middlewares/<Name>Middleware.ts tests/middlewares/<Name>Middleware.spec.ts
```
