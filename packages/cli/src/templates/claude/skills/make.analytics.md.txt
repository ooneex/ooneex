# Make Analytics Class

Generate a new analytics class and its test file using the `make:analytics` CLI command, then complete the generated code with proper implementation.

## Coding Conventions

- Always explicitly show visibility on class methods and properties (`private`, `public`, `protected`)
- Always use arrow functions when possible, but NOT for class methods (class methods use regular method syntax)
- End type names with the `Type` suffix (e.g., `ExceptionStackFrameType`, `ServiceDataType`)
- Start interface names with the `I` prefix (e.g., `IException`, `IService`, `IAiChat`)
- For Entity class, if property is optional, add `null` to its type.

## Steps

### 1. Run the generator

Run the following command to scaffold the analytics class and test files:

```bash
bunx @ooneex/cli@latest make:analytics --name <name> --module <module>
```

Where `<name>` is the name provided by the user. The `--module` option is optional â€” if provided, files are generated under `modules/<module>/` instead of the project root. The command will generate:
- `src/analytics/<Name>Analytics.ts` - The analytics class file (or `modules/<module>/src/analytics/<Name>Analytics.ts` with `--module`)
- `tests/analytics/<Name>Analytics.spec.ts` - The test file (or `modules/<module>/tests/analytics/<Name>Analytics.spec.ts` with `--module`)

### 2. Read the generated files

Read both generated files to understand the scaffolded code.

### 3. Complete the analytics class

Edit `src/analytics/<Name>Analytics.ts` to complete the implementation:

- Implement the `capture` method with actual analytics tracking logic
- Define a proper type for `CaptureOptionsType` instead of `Record<string, unknown>` based on the analytics purpose
- Add any additional methods or properties relevant to the analytics class purpose

The generated class structure follows this pattern:

```typescript
import { decorator } from "@ooneex/analytics";
import type { IAnalytics } from "@ooneex/analytics";

type CaptureOptionsType = Record<string, unknown>;

@decorator.analytics()
export class <Name>Analytics<T extends CaptureOptionsType = CaptureOptionsType> implements IAnalytics<T> {
  public capture(options: T): void {
    // console.log("Analytics captured:", options);
  }
}
```

### 4. Complete the test file

Edit `tests/analytics/<Name>Analytics.spec.ts` to add meaningful tests beyond the scaffolded ones:

- Keep the existing scaffolded tests (class name, capture method)
- Add tests relevant to the specific analytics class behavior

The generated test structure follows this pattern:

```typescript
import { describe, expect, test } from "bun:test";
import { <Name>Analytics } from "@/analytics/<Name>Analytics";

describe("<Name>Analytics", () => {
  test("should have class name ending with 'Analytics'", () => {
    expect(<Name>Analytics.name.endsWith("Analytics")).toBe(true);
  });

  test("should have 'capture' method", () => {
    expect(<Name>Analytics.prototype.capture).toBeDefined();
    expect(typeof <Name>Analytics.prototype.capture).toBe("function");
  });
});
```

### 5. Lint and format

Run linting and formatting on the generated files:

```bash
bunx biome check --fix src/analytics/<Name>Analytics.ts tests/analytics/<Name>Analytics.spec.ts
```
